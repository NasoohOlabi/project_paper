\section{Methodology}
\label{sec:methodology}

This research proposes a black-box steganographic technique that embeds information within the semantic architecture of a conversation. The methodology is predicated on a standard covert communication scenario involving Alice (the sender) and Bob (the receiver), communicating over a public social media platform. While this study utilizes Reddit as a primary case study, the underlying principles are platform-agnostic and transferable to any threaded discussion environment.

\subsection{System Architecture}
The core of the proposed framework centers on a multi-layer embedding process that leverages both the \textit{spatial location} of the response and the \textit{thematic subject} of the content as carriers for the secret payload. By mapping binary data to high-level conversational actions—specifically, the selection of a reply target and the determination of a tangent—the system maximizes channel capacity while preserving perceptual naturalness. In the implemented workflow, the receiver later reuses the published thread state and angle list to recover the payload.

\subsection{Operational Requirements}
To achieve synchronization, both the sender and receiver must maintain access to:
\begin{itemize}
    \item \textbf{Public Communication Channel:} Accounts on the target social media platform.
    \item \textbf{Shared Operational Parameters:} A deterministic search algorithm, post metadata, and predefined search constraints.
    \item \textbf{Synchronized LLM Configuration:} Access to an identical Large Language Model (or API) with a fixed \texttt{temperature=0} setting, ensuring deterministic output during the codebook construction phase.
\end{itemize}

\subsection{Search Determinism \& Synchronization Framework}
\label{sec:search_determinism}
Ensuring that both Alice and Bob operate on the same researched corpus is the cornerstone of reliable decoding. The implemented research stage is a reproducible preprocessing chain rather than an ad hoc web search. A search-term generator first extracts 12--20 unique queries from the post title, body, and URL using the workflow LLM at \texttt{temperature=0}; the resulting term list is cached in SQLite by post ID. Each query is then executed through the Google Custom Search backend with a fixed request shape (\texttt{first=1}, \texttt{count=10}), duplicate links and PDF results are removed, and the surviving URLs are fetched in batches of three. Content extraction uses a crawl4ai-based adapter that normalizes URLs, caches responses by SHA-256 hash, and falls back to a local HTTP endpoint when necessary. The resulting flat list of fetched page texts is stored as \texttt{search\_results}, and the pipeline records \texttt{search\_terms\_hash} and \texttt{search\_results\_hash} so the same corpus can be replayed by downstream stages.

\subsection{State Synchronization (Temporal Window)}
To ensure that both Alice and Bob visualize an identical state of the conversation tree, they must adhere to a negotiated \textit{Temporal Window Duration} ($T_{sync}$). This synchronization mechanism is operationalized to address the processing latency inherent in the sender's agent:
\begin{enumerate}
    \item \textbf{Sender Side:} Alice captures a snapshot of the discussion thread at time $t_{start}$, performs the multi-layer embedding, and subsequently publishes the generated stego-comment at $t_{publish}$.
    \item \textbf{Receiver Side:} Upon identifying the stego-contribution published at $t_{publish}$, Bob filters the thread's historical state to include only those comments established prior to $t_{publish} - T_{sync}$.
\end{enumerate}
The agreed-upon duration $T_{sync}$ ensures that the "Comment Chain" retrieved by the receiver precisely replicates the context available to the sender at $t_{start}$, even if new comments were published during the sender's computational cycle. This prevents state divergence and ensures that the decoder can confidently identify the exact parent comment and conversational state used for encoding.

\subsection{Payload Optimization \& Compression}
Prior to embedding, the raw payload undergoes a transformation layer designed to minimize bitstream length. We implement a \textit{dynamic dictionary-based compression} scheme that uses dynamic programming to minimize the total bit-cost:
\begin{enumerate}
    \item \textbf{Dictionary Synthesis:} A session-specific linguistic dictionary is synthesized from the post body, the flattened comment bodies, and the fetched \texttt{search\_results} text corpus.
    \item \textbf{Optimal Path Selection:} The compression engine scans payload substrings up to 250 characters and evaluates two encoding modes for each segment:
        \begin{itemize}
            \item \textit{Literal Encoding:} Encodes the raw UTF-8 bitstream with a short length header.
            \item \textit{Dictionary Encoding:} Encodes a reference to the dynamic dictionary using a tuple of \textit{(Dictionary Index, Offset, Length)}.
        \end{itemize}
    \item \textbf{Bit-Cost Optimization:} The dynamic program selects the sequence of encoding modes that minimizes the total compressed length. If the dictionary path is not shorter than standard UTF-8, the compressor falls back to the standard representation.
\end{enumerate}

\subsection{Two-Layer Semantic Embedding Architecture}

\subsubsection{Layer 1: Spatial Embedding (Reply Targeting)}
\label{sec:layer1}
The primary embedding layer determines the \textit{architectural placement} of the steganographic contribution.
\begin{enumerate}
    \item The system flattens the nested comment tree with a deterministic depth-first walk.
    \item The bit-width for this layer is dynamically calculated as $w_1 = \lceil \log_2(n + 1) \rceil$, where $n$ is the number of available comments.
    \item Index 0 corresponds to a top-level reply to the post, while any non-zero index selects a comment whose parent chain is reconstructed through \texttt{parent\_id} and \texttt{link\_id} lookups.
\end{enumerate}

\subsubsection{Layer 2: Intent-Based Embedding (Thematic Tangent Selection)}
\label{sec:layer2}
The secondary layer determines the \textit{communicative intent}.
\begin{enumerate}
    \item \textbf{TangentsDB Construction:} As detailed in Section \ref{sec:search_determinism}, the system constructs a serialized list of potential thematic directions (\textit{Tangents}).
    \item \textbf{Dynamic Bit-Mapping:} The bit-width for this layer is calculated from the total number of tangents, and the flattened list is assigned sequential \texttt{idx} values.
    \item \textbf{Thematic Selection:} The corresponding payload bits are mapped to a specific tangent within the serialized list. In the generation step, the first four candidates are used to build the primary prompts and few-shot validation examples, while each angle is paired with the best matching research text selected from \texttt{search\_results}.
\end{enumerate}

\subsection{Generative Synthesis \& Verification}
Following the selection of the target context and thematic tangent, the sender initiates the cover text synthesis process.
\begin{itemize}
    \item \textbf{Synthesis:} A \texttt{gpt-oss-20b} model configured with \texttt{temperature=0.7} constructs one JSON array of exactly three short Reddit-style comments. The prompt combines the source post metadata, the selected comment chain, the target tangent, and the best-match research text, and the parser accepts both raw JSON and fenced JSON responses.
    \item \textbf{Verification Loop:} Before the stego-text is committed to the public channel, the sender executes a local \textit{Decoding Simulation} to verify that the generated text remains recoverable.
    \item \textbf{Recursive Refinement:} If the simulation fails to recover the intended angle, the sender re-executes the synthesis step without changing the embedding parameters. The non-zero temperature ensures output variation; the retry loop is capped at five total attempts.
\end{itemize}

\subsection{Decoding \& Intent Recovery}
The receiver reuses the published angle list rather than rebuilding the corpus from scratch. To recover the hidden intent (Layer 2):
\begin{enumerate}
    \item \textbf{Semantic Filtering:} The receiver employs a semantic similarity model to perform a lookup against the pool of potential Tangents, identifying the top 20 candidates that most closely match the observed stego-text.
    \item \textbf{Discriminative Identification:} A specialized LLM prompt is executed on these candidates to achieve precise identification:
          \begin{quote}
              \textit{"Identify the specific tangent from the provided candidates... return the index only."}
          \end{quote}
\end{enumerate}
The decoder uses a \texttt{gpt-oss-20b} model at \texttt{temperature=0}. If the model returns a 1-based rank rather than a raw index, the pipeline maps the rank back to the shortlisted candidate; otherwise it falls back to the top semantic match. This high-precision recovery mechanism ensures robust extraction of the hidden payload with minimal error rates.
